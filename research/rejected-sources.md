# Rejected Sources

Sources that were evaluated and rejected, with reasons.

## Why Track Rejected Sources?
To avoid re-evaluating the same low-quality sources and to document quality decisions.

| Date | Source | Type | Reason Rejected |
|------|--------|------|-----------------|
|        |        |      |                 |

## Common Rejection Reasons
- `hype-only` — no concrete workflow, just promotion
- `tutorial-only` — installation guide, no production use
- `skill-no-users` — skill exists but no evidence of real usage
- `feature-request` — requested but not implemented
- `no-workflow` — no concrete workflow described
- `duplicate` — variant of existing family
- `low-evidence` — single unverified source
- `out-of-scope` — not related to OpenClaw use cases
